🔑 COMO FUNCIONA A AUTENTICAÇÃO
───────────────────────────────────────────────────────────────────────────────

FLUXO DE LOGIN:
  1. Usuário entra no app
  2. LoadingScreen verifica: tem login salvo?
  3. SIM → vai direto para HomeScreen (sem pedir login novamente)
  4. NÃO → mostra LoginScreen com botão "Entrar com o Google"
  5. Usuário clica → login simulado (depois será real via Google)
  6. Dados salvos localmente com AsyncStorage
  7. HomeScreen mostra mensagem: "Seja bem-vindo ao App que vai te..."

FLUXO DE LOGOUT:
  1. Na HomeScreen, clica em "Sair"
  2. Deleta dados salvos
  3. Volta para LoginScreen


📝 ARQUIVOS PRINCIPAIS E O QUE FAZEM
───────────────────────────────────────────────────────────────────────────────

  └─ Renderiza <AuthProvider> + <AppNavigator>
  └─ AppNavigator mostra LoadingScreen → LoginScreen → HomeScreen conforme estado

src/context/AuthContext.js
  └─ useAuth() — hook para acessar { user, loading, signIn, signOut }
  └─ loadStoredUser() — verifica AsyncStorage ao abrir o app
  └─ signIn(userData) — salva usuário localmente
  └─ signOut() — apaga dados e desautentica

src/screens/LoginScreen.js

  └─ Botão "Entrar com o Google"

src/screens/HomeScreen.js

🔐 PRÓXIMOS PASSOS: LOGIN REAL COM GOOGLE
───────────────────────────────────────────────────────────────────────────────

Para ativar o login REAL com Gmail, você precisa:

1. Google Cloud Console
   • Vá para: https://console.cloud.google.com
   • Crie um novo projeto
   • Vá em: APIs e Serviços → Credenciais
   • Crie um cliente OAuth para Android + Web

2. SHA-1 do seu keystore
   • Terminal: cd android && ./gradlew signingReport
   • Copie o SHA-1 no Google Cloud Console

3. Baixar google-services.json
   • Google Cloud Console → Download JSON
   • Salve como: C:\Sandro\PY\Sandro\Meu Treino\google-services.json

4. Ativar no código
   • Abra: src/screens/LoginScreen.js
   • Procure por: "SEU_WEB_CLIENT_ID_AQUI"
   • Substitua pelo Web Client ID do Google
   • Descomente a seção "LOGIN REAL COM GOOGLE"

5. Recompile
   • npx expo start (talvez precise limpar cache)

💡 DICAS PARA DESENVOLVIMENTO
───────────────────────────────────────────────────────────────────────────────

🔴 Quando mudar código:
  • Salve e o Expo recompila automaticamente no celular (Fast Refresh)
  • Se der erro, força: Pressione 'r' no terminal

🔴 Debug:
  • Abra Developer Menu no celular (shake device ou Cmd+D no emulador)
  • Escolha "Debug remote JS"
  • DevTools abrem no PC

🔴 Dados persistidos:
  • AsyncStorage salva na memória do celular
  • Para limpar: dev menu → "Clear AsyncStorage"
  • Ou use: await AsyncStorage.clear()

🔴 Testes de velocidade:
  • Crie usuários diferentes para testar fluxo completo

🔴 Ícones:
  • Atualmente usa emojis (💪, 🏋️, etc)
  • Futuro: trocar por Tabler Icons (ti-heart, ti-star, etc)


❓ PERGUNTAS FREQUENTES
───────────────────────────────────────────────────────────────────────────────

P: Como deletar o login salvo para testar de novo?
R: No celular, abra Expo Go → seu projeto → shake → Clear AsyncStorage

P: Posso rodar no iOS?
R: Sim! Expo Go funciona em iOS também. Mesmo QR Code.

P: Preciso de Android Studio?
R: Não para testar com Expo Go. Só se quiser buildar APK/AAB.

P: Como faço build para distribuir?
R: eas build --platform android (requer conta Expo)

P: Posso usar meu IP da rede local?
R: Sim! No terminal, escolha a opção de rede local ao invés de Tunnel.

P: O app funciona offline?
R: Sim! Após primeiro login, funciona sem internet (dados salvos localmente).
